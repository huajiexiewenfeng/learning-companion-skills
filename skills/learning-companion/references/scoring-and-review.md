# Scoring And Review

The skill tracks both plan progress and effective progress.

## Mastery Scale

```text
0/5 = did not study
1/5 = touched the material but cannot state the core idea
2/5 = can repeat material words but cannot explain the use
3/5 = can state the use in their own words and answer at least one verification question
4/5 = can explain scenario, misconception, boundary, or key distinction
5/5 = can transfer the idea to a real problem, case, writing, or project judgment
```

Default effective completion threshold: `3/5`.

## Subject-Agnostic Verification

Choose verification questions by subject:

- Technology: role, scenario, distinction, common mistake.
- Literature: theme, character motive, textual detail, interpretation.
- Philosophy: core thesis, argument structure, objection, example.
- Language: recall, sentence creation, paraphrase, small output.
- Professional skills: situation, decision rule, trade-off, application.

Do not turn verification into a long exam.

## Progress Strategy

```text
0-2: rescue, pause new progress
3: complete, next session starts with light review
4-5: complete, normal progress
```

## Rescue Tasks

Rescue tasks should return to the user's original material.

Do:

- point back to the current topic or source section
- reduce the task to 10-30 minutes
- ask for one concrete understanding output

Do not:

- replace the source material
- add a long external lecture
- redesign the curriculum without permission

## Weekly Review

Run per plan.

Include:

- plan progress change
- effective progress change
- study days
- effective completion days
- low-power days
- skipped days
- missed close-outs
- repeated risks or stuck points
- suggested next-week strategy

Weekly stage reorder requires user confirmation before writing changes.

Daily rescue can be applied automatically. Weekly stage reorder is proposed first, then applied only after the user confirms.
