# North Star Workflow

Use this reference when the learner's goal is vague, broad, or mixed across several possible outcomes.

## Purpose

A North Star is the course's decision anchor.

It answers:

```text
What future capability should this course create?
```

Do not treat a topic as a North Star.

```text
Weak: Learn AI
Better: Build an enterprise AI transformation architecture from RAG to governed autonomous operations.

Weak: Learn Java
Better: Become able to build and explain production-grade Java backend services for Agent Runtime systems.
```

## One-Question-At-A-Time Discovery

Ask only the highest-leverage question needed.

Useful questions:

- What do you want to be able to build, publish, explain, or do after this course?
- Is the goal for career transition, current work, portfolio, business product, or personal interest?
- What real project, job role, or output should the course connect to?
- How much time can you reliably study per week?
- What would make you say "this course worked" after 3-6 months?

Avoid long intake forms unless the user explicitly asks for a full questionnaire.

## North Star Test

A strong North Star should pass most of these checks:

- It names a future capability, not only a topic.
- It can guide what to include and exclude.
- It connects to the user's real context.
- It can produce visible outputs.
- It can be decomposed into stages.
- It supports verification.

## Example From Enterprise AI Course

Raw idea:

```text
I want to learn RAG, Role Agent Copilot, Agentic Workflow, AI Native App, and Autonomous Operation.
```

North Star:

```text
Complete an end-to-end enterprise AI transformation architecture path:
from trusted knowledge runtime, to role copilots, to agentic workflows,
to AI-native applications, to governed autonomous operations.
```

Course design implication:

- Do not study RAG as an isolated demo.
- Connect every RAG concept to Role Copilot and governance.
- Use a spiral course where all layers appear early, then deepen over time.
- Require visible outputs every sprint.

## When The User Has Multiple North Stars

If the user mixes several outcomes, separate them:

```text
Career goal:
Portfolio goal:
Project goal:
Learning goal:
Content/publishing goal:
```

Then recommend one primary North Star and 1-2 secondary constraints.

Do not design a course that optimizes equally for every possible goal.
